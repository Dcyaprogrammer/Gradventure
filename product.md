# Description
Gradventure is:
a web applicaton
a rogue-like game
following the design as the game reigns
The users will be engaged in a rogue-like game and learn application related knowledge. Users can also gain credits along the process and trade for real-life rewards.
Since the knowledge are highly distributed during the game, we will develop a "wiki" to store the potential knowledge classified.

The game will go from year1 to year3 and users will encounter different events and have to make their choices to balance status(metrics), the currently decided metrics are GPA, Mentality, Energy and Experience 

The content is expected to be from database in the future, but for this development stage, we will hardcode the content in the game page.

The ultimate goal of the development is to develop a scalable gaming system, the demonstration will be only required for a demo scale.

# Components
1. Home page
2. Login page
3. Game page(Cards, status etc.)
4. Shop page(Trade for real-life rewards)
5. Profile page(Show user info and credits)


# Frontend
1. Neo-brutalism
2. Extremely various animations/interactions
3. Responsive design
4. No learned design patterns
5. Be bold on color choice but keep it clean
6. Components should be varified, not like single circles or squares are used for all components
7. refer to the neo-ne-brutalism-ui-library in the root directory
8. We all mainly developing for mobile phone users, so the design should be mobile-friendly
